# Planning Templates

These templates are the starting point for new roadmap and phase-planning work in `mako-ai`.

Use them when:

- starting a new roadmap generation
- planning a follow-on phase after the initial roadmap
- handing a scoped implementation brief to another agent

Do not treat the templates themselves as source of truth. Copy them into the real target doc, then fill them with project-specific decisions, constraints, and done-when checks.

Available templates:

- [roadmap-template.md](./roadmap-template.md) - canonical roadmap structure
- [phase-implementation-plan-template.md](./phase-implementation-plan-template.md) - concrete phase-spec structure

Recommended workflow:

1. Start with the roadmap template when the work changes product shape or phase order.
2. Use the phase implementation template when one phase needs a concrete execution brief.
3. Keep the roadmap high-level and canonical.
4. Keep phase specs concrete, file-aware, and testable.
